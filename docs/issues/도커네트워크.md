#Docker 컨테이너 간 PostgreSQL 연결 문제 해결 리포트

## 1. 문제 상황 (Issue)
증상: Spring Boot 서버 컨테이너 실행 시 PostgreSQL 연결 실패.
에러 로그: java.net.UnknownHostException: some-postgres
원인: 서버 컨테이너가 PostgreSQL 컨테이너의 호스트명(some-postgres)을 IP로 해석하지 못함. (도커 기본 네트워크의 DNS 제한 문제)

## 2. 진단 (Analysis)
네트워크 분리: docker inspect 결과, DB 컨테이너는 특정 네트워크(172.19.0.2)에 속해 있었으나 서버 컨테이너는 해당 네트워크에 참여하지 않아 격리된 상태였음.
포트 혼동: 호스트 OS에서 사용하는 포트(5433)와 도커 내부 통신용 포트(5432)의 차이 확인 필요.

## 3. 해결 방법 (Solution)
### 단계 1: 컨테이너를 동일 네트워크로 통합
도커 내부 DNS 기능을 활용하기 위해 두 컨테이너를 같은 사용자 정의 네트워크(Custom Network)에 연결했습니다.


```Bash
# 1. 네트워크 연결 (DB가 속한 네트워크에 서버를 추가) 네트워크이름은 mcp-net 으로 설정했음.
docker network connect [네트워크_이름] mcp-server

# 2. 서버 컨테이너 재시작 (네트워크 설정 적용)
docker restart mcp-server

```

### 단계 2: 어플리케이션 설정 수정
application.yml 또는 properties 파일의 접속 정보를 도커 내부 통신 규격에 맞게 설정했습니다.
즉 내부 연결은 뒤 포트를 사용.

기존 (오류 가능성): localhost:5433 또는 some-postgres:5433
변경 (성공): jdbc:postgresql://some-postgres:5432/db
Host: 컨테이너 이름 (some-postgres)
Port: 도커 내부 포트 (5432)


## 4. 핵심 요약 (Key Learnings)
Internal vs External Port: 5433:5432 설정 시, 외부(DBeaver, IntelliJ 등)에서는 5433을 쓰고, **도커 컨테이너끼리는 5432**를 써야 합니다.

Docker DNS: 컨테이너 이름으로 서로를 찾으려면 반드시 **사용자 정의 네트워크(Custom Network)**를 생성하고 그 안에 함께 속해 있어야 합니다.

Connectivity Check: docker inspect를 통해 컨테이너가 어떤 네트워크 대역(IPAddress)을 할당받았는지 확인하는 것이 문제 해결의 시작입니다.
